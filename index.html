In the past, non-semantic elements like 'div' or 'span' were used to contain or group code together. These elements do not provide the user or browser any guide as to what the element is used for or how it fits in the structure of the page. Although classes, ids and attributes allow us to provide some definition of the element, each non-semantic element must be interpreted to determine what is contained within the element. As HTML evolved, semantic elements were defined to have a pre-defined purpose. Instead of using the generic 'div' element, semantic elements like 'header', 'nav', 'aside', 'footer' and others, have a specific use and meaning that can be easily identified and interpreted by the developer or browser.
